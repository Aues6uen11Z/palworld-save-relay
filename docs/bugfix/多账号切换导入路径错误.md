# Bug: 多 Steam 账号切换后导入/导出操作指向错误目录

## 版本
v0.5.12

## 现象
多 Steam 账号环境下，房主 A 的 Steam 号点导出后，切到 B 的 Steam 号点导入，实际文件导入到了 A 的目录而非 B 的，B 需要手动覆盖才行。

## 原因
`refreshWorlds()` 在获取新世界列表后，仅在 `selWorld` 为 null 时自动选择第一个世界。切换 Steam 账号时 `selWorld` 不为 null（仍指向 A 的路径 `saveRoot/SteamID_A/GUID`），世界列表更新为 B 的世界但 `selWorld` 未更新。用户点导入时操作的是 A 的目录。

## 修复
`refreshWorlds()` 获取新世界列表后，按 GUID 匹配当前选中的世界：
- 新账号下有同 GUID 世界且路径不同 -> 更新 `selWorld` 到新路径
- 新账号下无同 GUID 世界 -> 自动选第一个世界
- `selWorld` 为 null -> 自动选第一个世界
