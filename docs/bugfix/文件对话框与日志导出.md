# Bug: 文件选择框取消报错 + 云服务提示框烦人 + 缺少导出日志功能

## 版本
v0.5.12

## 现象
1. 点导入打开文件选择框，取消时会报错 `Invalid dialog call: Dialog.OpenFile failed, error getting selection: cancelled by user`
2. 一直不打算配云服务的用户，每次都看到"还没配置云服务"的黄色提示框
3. 排查问题时需要用户手动找日志文件，不方便

## 原因
1. `Dialogs.OpenFile` / `Dialogs.SaveFile` 取消时抛出异常，catch 块直接 `flash("err", ...)` 显示
2. `needsConfig` 条件渲染的黄色提示框对不使用云服务的用户无意义
3. 无导出日志的后端方法和前端入口

## 修复
1. catch 块增加 `cancelled by user` 正则检测，匹配则静默忽略
2. 删除黄色提示框及不再使用的 `needsConfig` 变量
3. 后端新增 `ExportLog(outPath)` 方法，复制 `%APPDATA%/PalSaveRelay/app.log` 到指定路径
4. 前端设置页「关于」区域新增「📄 导出日志」按钮，弹出保存对话框选择位置

## 修复 Commit
`254e3e1` feat: 错误提示i18n + 日志可诊断性 + 多Steam账号切换bug修复

## 涉及文件
- `frontend/src/App.tsx`: 取消检测 + 删除提示框 + 导出日志按钮
- `app.go`: `ExportLog` 方法
- `frontend/src/i18n.tsx`: `settings.exportLog` / `toast.logExported` 条目
