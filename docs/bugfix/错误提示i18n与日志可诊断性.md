# Bug: 错误提示为原始 Go 错误串 + 日志缺乏操作关联

## 版本
v0.5.12

## 现象
- 操作失败时弹出的错误提示是原始 Go error 字符串（如 `qiniu config incomplete`），中文 UI 下也是英文，用户无法理解
- 日志中同一操作的多个步骤没有关联标识，用户发来日志后难以快速定位是哪次操作出了问题

## 原因
- 后端直接返回 `error`，前端用 `String(e?.message || e)` 拼接显示，无结构化错误码
- 日志各条目只有函数名 + world GUID，多次操作交叉时无法区分

## 修复
1. 新增 `internal/apperr` 包，定义 18 个错误码（`GAME_RUNNING`、`QINIU_CONFIG`、`VALIDATION_FAILED`、`PACK_FAILED`、`STRIP_FAILED`/`STRIP_FATAL`、`REPLACE_FAILED`/`REPLACE_FATAL` 等）。`CodedError.Error()` 输出 `[CODE] detail` 格式
2. `app.go` 所有关键错误路径套上对应错误码
3. `app.go` 新增 `opID()` 自增操作 ID，每次操作从 start 到 done 的日志都带同一 `#000001` 格式 ID
4. repair 日志补全 builders/slots/ownerless 修复计数
5. 前端 i18n 新增 18 条 `errcode.*` 条目（中英双语）
6. 新增 `parseErr(e, t)` 提取 `[CODE]` 并映射为本地化消息，未知 code 去掉前缀返回 detail
7. `App.tsx` 所有错误展示改用 `parseErr` 替代原始错误字符串
