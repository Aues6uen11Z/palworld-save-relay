# Palworld 存档转换 需求文档

## 一、项目背景

Palworld 联机模式存档仅保存在房主本地，房主离线后其他玩家无法继续游戏。本工具实现「存档接力」，让多人轮流当房主，共享同一个世界进度。

## 二、项目目标

- 做一款**单文件、零依赖**的桌面工具，普通玩家开箱即用
- 基于七牛云对象存储做存档中转，国内访问速度快、成本低
- 房主切换全自动，用户无需了解 UID、存档结构等技术细节

## 三、核心功能

### 3.1 房主切换（核心）
- 自动检测本地 Palworld 存档目录，支持隐藏/展示存档（选出共同游玩的存档），支持给存档起别名（不改实际文件名）
- 自动扫描存档内玩家列表，用户选择「我是谁」
- 一键将指定玩家切换为房主（UID 全局替换）
- 切换前自动备份，支持一键回滚

### 3.2 云存档同步
- 七牛云对象存储对接（AccessKey / SecretKey / Bucket / 区域 四项配置）
- 一键上传当前存档到云端
- 一键下载最新存档到本地
- 云端版本历史列表（最近 N 个版本）
- 简单游玩锁机制，提示当前是否有人在玩

### 3.3 导入导出（兜底）
- 导出存档为单文件，支持手动传输
- 导入外部存档文件，自动切换房主后写入本地

### 3.4 备份管理
- 本地自动备份（每次切换前），保留最近 5 份
- 备份列表查看、回滚、删除

## 四、用户角色与主流程

### 角色
- **接力玩家**：同一世界的多个玩家，轮流担任房主，共用工具

### 主流程
1. 玩家 A 游玩结束 → 点击「上传存档」
2. 玩家 B 打开工具 → 点击「下载最新存档」→ 点击「切换为我房主」→ 启动游戏
3. 玩家 B 游玩结束 → 上传存档，完成接力

## 五、技术栈

| 层级 | 选型 |
|------|------|
| 桌面框架 | Wails v3 |
| 后端语言 | Go 1.22+ |
| 前端 | Vue 3 + Vite |
| 云存储 | 七牛云对象存储 Kodo |
| 打包输出 | Windows 单 exe |

## 六、非功能需求

- **数据安全**：修改存档前必备份，操作可回滚，不坏档
- **网络兼容**：国内网络稳定上传下载，支持断点续传
- **易用性**：首次使用引导，三步以内完成接力操作
- **可扩展**：存储层抽象，后续可接入 OSS / COS 等其他后端
